# AGENTS.md

Guidance for AI coding agents working in this repository.

## Scope

- This repo is a TypeScript ESM library that exports flat ESLint config presets.
- Use `pnpm` (workspace uses `pnpm@11.0.8`).
- Node version must satisfy `^20.19.0 || ^22.13.0 || >=24`.

## Fast Start

- Install: `pnpm install`
- Build library: `pnpm build`
- Dev watch build: `pnpm dev`
- Lint: `pnpm lint`
- Format: `pnpm format` (uses `oxfmt`, not Prettier)
- Test: `pnpm test`
- Typecheck: `pnpm typecheck`
- Full release validation: `pnpm release:check`
- Docs dev/build: `pnpm docs:dev` / `pnpm docs:build`

## Architecture Map

- `src/core.ts`: `defineESLintConfig()` factory that composes all config modules.
- `src/configs/*.ts`: one config module per feature (`configXxx` + options type).
- `src/configs/index.ts`: barrel exports for all config modules.
- `src/types/config.ts`: top-level `ConfigOptions` contract.
- `src/types/options.ts`: shared option types reused by config modules.
- `src/globs.ts`: centralized glob constants; prefer adding/updating globs here rather than inline patterns.
- `src/utils/env.ts`: package-detection helpers (`hasVue`, `hasTypeScript`, etc.).
- `src/utils/ensurePackages.ts`: optional interactive dependency installer (auto-skips in CI/non-TTY/hooks).
- `src/eslint/*`: parsers/plugins/processors/config wrappers and re-exports.

## Non-Obvious Rules

- Do not manually edit `src/types/typegen.d.ts`; it is generated by `scripts/generateType.ts`.
- If you change config composition or rule sets, run `pnpm types:generate` before final checks.
- Keep config-module naming consistent: `configXxx` and `ConfigXxxOptions`.
- Preserve ESM style (`"type": "module"`); avoid introducing CommonJS patterns.
- Keep changes focused; avoid broad reformatting unrelated files.

## Change Workflow

When modifying config logic:

1. Update implementation in `src/configs/*.ts` and/or `src/core.ts`.
2. Update related docs in `docs/configs/*.md` (same feature area).
3. Regenerate types: `pnpm types:generate`.
4. Validate: `pnpm lint && pnpm test && pnpm typecheck`.

For release-level confidence, run `pnpm release:check`.
